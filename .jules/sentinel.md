## 2025-02-14 - Fix XSS Vulnerability in ImpactHeader

**Vulnerability:** The component `ImpactHeader` was using `dangerouslySetInnerHTML` to inject a styled `<span>` for the keyword 'transforma' from a `title` prop. This allows any arbitrary string with HTML tags to be directly injected into the DOM, making it vulnerable to XSS.
**Learning:** This occurred because replacing text in a string directly using `.replace()` to append HTML elements requires executing the resulting HTML using `dangerouslySetInnerHTML`. However, doing so unsafely is risky if `title` comes from user input. Moreover, we learned from memory that using `<React.Fragment>` to build arrays in this codebase crashes Next.js SSR apps, and `isomorphic-dompurify` also has issues due to `jsdom` errors.
**Prevention:** Rather than directly manipulating HTML strings to do text highlights, we can split strings using a regex with a capture group (e.g., `title.split(/(transforma)/i)`). This correctly keeps the matched string intact in the resulting array. Then we can `.map()` the array, styling the matched strings directly as JSX nodes and returning non-matching strings as plaintext, without exposing the app to XSS.
