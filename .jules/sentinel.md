## 2024-05-18 - [HIGH] XSS Vulnerabilities in UI Components (dangerouslySetInnerHTML)
**Vulnerability:** Several UI components (`NewsCard.tsx`, `PostCard.tsx`, `ImpactHeader.tsx`) were using React's `dangerouslySetInnerHTML` directly with unsanitized content (like CMS strings or simple text replacements) which can lead to Cross-Site Scripting (XSS) attacks.
**Learning:** In Next.js environments, typical libraries like `isomorphic-dompurify` can cause SSR errors due to `jsdom` ENOENT. As a result, `sanitize-html` is the preferred library. Additionally, for simple keyword highlighting, using `String.prototype.split()` and React mapping (with `<span>` elements, avoiding `<React.Fragment>` to prevent JSX runtime issues) provides a much safer alternative to string interpolation and raw HTML setting.
**Prevention:** Always sanitize strings originating from CMS/external sources with `sanitize-html` before passing them to `dangerouslySetInnerHTML`. Where possible, rely on string splitting and safe React component mapping over setting inner HTML.
