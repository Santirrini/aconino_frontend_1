## 2025-04-11 - [HIGH] XSS Vulnerability in Text Highlighting via dangerouslySetInnerHTML
**Vulnerability:** A cross-site scripting (XSS) vulnerability was found in `ImpactHeader.tsx` where user-controlled input (`title`) was being rendered using `dangerouslySetInnerHTML` after performing a simple string replacement for keyword highlighting (`title.replace('transforma', '<span>...</span>')`).
**Learning:** This existed because standard React string rendering does not support embedding HTML directly from a string replace operation without using `dangerouslySetInnerHTML`. The developer attempted to add styling to a specific keyword within a dynamic string. This highlighted the architectural gap of lacking a safe utility for highlighting keywords in strings.
**Prevention:** To avoid this in the future and safely highlight specific keywords within a string without using `dangerouslySetInnerHTML`, use `String.prototype.split()` with a capturing group regex (e.g., `/(keyword)/i`) and map the resulting segments to React fragments or styled spans. This approach leverages React's built-in XSS protection while allowing targeted styling.
